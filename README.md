## 前置知识

### 什么是情感二分类

情感二分类的目标是：**给定一条中文电商评论，判断它是好评还是差评**。

例如：
- "产品质量很好，做工精细，物流也很快，非常满意！" → **好评**
- "用了不到一个月就坏了，客服也不理人，太失望了。" → **差评**
- "一般般吧，没想象中那么好用，凑合着用。" → **差评**（虽无明显负面词，但整体满意度偏低，星级映射到差评）

计算机只理解数字，需要把文字变成数字、再通过这些数字做分类。不同的模型用不同的方式完成这个任务——这就是整个项目的核心。

### TF-IDF 是什么

TF-IDF（Term Frequency — Inverse Document Frequency，词频—逆文档频率）是传统机器学习处理文本的**基础输入特征**，用来衡量"一个词对一条评论有多重要"。

**核心思想**：不是所有词都同样重要。比如"的""了""产品"几乎每篇评论都有，对区分好评差评没什么用；而"太差了""非常满意"只在特定类别的评论中出现，才是真正有区分力的词。TF-IDF 就是对前者降权、对后者升权。

**计算公式**：

```
TF-IDF(词, 评论) = TF(词, 评论) × IDF(词)
```

- **TF（词频）** = 该词在这条评论中出现的次数 / 这条评论的总词数。表示该词在这条评论里有多"密集"。
- **IDF（逆文档频率）** = log(总评论数 / 包含该词的评论数)。表示该词在整个语料中有多"稀缺"。如果几乎每篇都有这个词，IDF 接近 0；如果只有少数评论出现，IDF 很大。

**举例**：假设语料中有 10000 条评论：

| 词 | 出现在多少条 | IDF | TF-IDF | 含义 |
|---|---|---|---|---|
| "的" | 9800 | ≈0.02 | **极低** | 几乎没区分力 |
| "产品" | 8000 | ≈0.22 | **低** | 太通用 |
| "太差了" | 300 | ≈3.5 | **高** | 有区分力的情感信号 |

最终每条评论被表示为一个 **5000 维的稀疏向量**（大多数值为 0，因为一条评论不可能用到词表中的所有词），每一维是该评论中对应词的 TF-IDF 值。这个向量就是 MultinomialNB、LinearSVC、XGBoost 三个传统模型的共同输入。

## 数据均衡

`01_sampling.py` 默认合并 `训练集.csv` + `online_shopping_10_cats.csv`（共 132,773 条），通过**下采样**好评使正负比例达到 1:1（均衡后约 88,908 条）。

**为什么要均衡？** 电商评论中好评通常占 80%+，如果不均衡，模型会偏向预测好评（只要全猜好评就能拿 80% 准确率，但差评的识别能力接近 0）。下采样多数类是解决这个问题最简单有效的方法。

如果使用其他未均衡的数据，可通过 `--balanced` 启用各脚本的类权重机制：

```bash
python scripts/02_train_traditional.py --balanced
python scripts/03_train_dl.py --balanced
python scripts/04_train_pretrained.py --balanced
```

各方法的 `--balanced` 对应策略：

| 方法 | --balanced 启用时 |
|------|-----------|
| LinearSVC | `class_weight='balanced'` — 少数类的 hinge loss 加重惩罚 |
| XGBoost | `compute_sample_weight('balanced')` — 少数类样本的残差放大 |
| TextCNN / BiGRU | WeightedRandomSampler（少数类更频繁采样）+ 加权 CrossEntropyLoss |
| BERT / RoBERTa | 加权 CrossEntropyLoss — 少数类分类错误的损失更大 |

**主要评价指标**：**Macro-F1**（宏平均 F1）。它是对差评 F1 和好评 F1 做简单算术平均，不会因为某个类别样本多就被拉偏。在均衡数据下，Macro-F1 公平地衡量模型在两类上的综合表现。

## 使用流程

### Step 1: 准备数据

将原始数据（CSV / TSV / JSON / JSONL）放入 `data/raw/`，确保包含评论内容列和评分列。

脚本会自动识别两种格式：
- **星级格式**：包含"评论内容"/"review"等文本列 + "评分"/"star"等星级列（通过 `star_to_label()` 转换）
- **标签格式**：包含"review"/"comment"等文本列 + "label"标签列（0/1，直接使用）

### Step 2: 数据采样

```bash
# 默认合并两个数据集，均衡后划分
python scripts/01_sampling.py

# 只用单个数据集
python scripts/01_sampling.py --input data/raw/训练集.csv

# 不均衡（保留原始分布）
python scripts/01_sampling.py --no-balance
```

脚本处理流程：加载各文件 → 检测格式 → 星级转二分类 → 数据清洗（去空、去无效标签） → 合并 → 下采样多数类至 1:1 → 分层抽样划分为 `train.csv`、`val.csv`、`test.csv`（8:1:1）。

### Step 3: 训练模型

**传统机器学习（TF-IDF + NB / SVM / XGBoost）**：

```bash
# 默认训练（TF-IDF 5000 维，XGBoost 默认超参）
python scripts/02_train_traditional.py

# 调参模式：char+word 联合特征 + RandomizedSearchCV（推荐）
python scripts/02_train_traditional.py --tune-xgboost
```

**深度学习从头训练（TextCNN + BiGRU-Attention）**：

```bash
# 训练全部（推荐带预训练词向量）
python scripts/03_train_dl.py --wv-path data/embeddings/sgns.target.word-ngram.1-2.dynwin5.thr10.neg5.dim300.iter5

# 只训练一个
python scripts/03_train_dl.py --model textcnn
python scripts/03_train_dl.py --model bigru_attn

# 不使用预训练词向量（随机初始化，效果稍差但不需额外文件）
python scripts/03_train_dl.py
```

**微调预训练模型（BERT + RoBERTa）**：

```bash
python scripts/04_train_pretrained.py --model all      # 训练全部
python scripts/04_train_pretrained.py --model bert     # 只训练 BERT
python scripts/04_train_pretrained.py --model roberta  # 只训练 RoBERTa
```

### Step 4: 汇总结果

```bash
python scripts/05_evaluate_all.py
```

读取 `outputs/results.csv`，按 Macro-F1 排序打印所有模型对比，并生成：
- `outputs/figures/model_comparison.png` — 各模型 Accuracy / Macro-F1 / Weighted-F1 对比柱状图
- `outputs/figures/per_class_f1.png` — 各模型在差评/好评上的 F1 对比

### Step 5: 启动 Web Demo

项目提供两种前端界面：

**方式一：FastAPI 桌面版（推荐）**


基于 FastAPI + 独立 HTML/CSS 前端，桌面优化双栏布局，暖调配色（象牙白底 + 琥珀金点缀）：
- **模型选择下拉框**：在全部 7 个模型间实时切换，显示各模型的 Macro-F1 / Accuracy 徽章
- **单条预测**：输入评论，显示带带动画进度条的情感结果卡片

## 模型详解

本项目的 7 个模型可以分为三大类：

| 类别 | 模型 | 核心思路 |
|------|------|---------|
| 传统机器学习 | MultinomialNB, LinearSVC, XGBoost | 基于 TF-IDF 数值特征做统计分类 |
| 深度学习从零训练 | TextCNN, BiGRU-Attention | 用神经网络从词序列中自动学习语义模式 |
| 预训练模型微调 | BERT, RoBERTa | 加载已在海量中文语料上训练好的"通才"模型，针对我们的任务做微调 |

下面逐个详解每个模型。

---

### 1. MultinomialNB（多项式朴素贝叶斯）

#### 是什么

多项式朴素贝叶斯是一种基于**贝叶斯定理**的统计分类方法。它假设每条评论是一堆从"词袋"里独立抽出来的词，只不过不同的类别（好评/差评）有各自不同的词概率分布。

"多项式"指文档中每个词的出现次数遵循多项式分布，"朴素"指假设词与词之间是条件独立的（这是一个被刻意简化的假设——实际上"非常"和"满意"显然相关——但有了这个假设后计算变得极简单，实践中效果仍然不错）。

#### 为什么用在这个项目

- **训练极快**（秒级）：只需要统计计数，无需迭代优化
- **可解释性强**：可以看到每个词对预测的贡献
- **在短文本上效果尚可**：作为项目的 baseline，提供一个"如果什么高级方法都不用，最低能做到多好"的参照线

#### 如何训练的

TF-IDF 词频向量（5000 维）直接作为输入。

内部做的事很简单：

**第一步：统计先验概率** P(类别)。由于数据均衡处理过，P(差评) = P(好评) = 0.5。

**第二步：计算似然概率** P(词|类别)。统计每个词在各个类别中出现的频率：

```
P(词|类别) = (该词在类别中出现的总次数 + α) / (该类别所有词的总次数 + α × 词表大小)
```

这里的 **α = 0.5** 称为拉普拉斯平滑参数。它的作用是防止某个词在差评中的次数为 0 时，整个概率乘积直接变为 0（一个词没见过就否定全句显然不合理）。α=0.5 比 sklearn 默认的 1.0 更小，意味着更信任训练数据本身的统计。

**第三步：预测新评论**。对于一条新评论，计算它属于两个类别的后验概率，取对数防止浮点数下溢（几百个 0~1 的小数相乘会指数级缩小到计算机无法表示）：

```
log P(差评|评论) ∝ log P(差评) + Σ TF-IDF(w) × log P(w|差评)
log P(好评|评论) ∝ log P(好评) + Σ TF-IDF(w) × log P(w|好评)
```

哪边大预测哪边。TF-IDF 在这里替代了原始词频——关键情感词（如"太差了"）TF-IDF 高，对结果影响大；通用词（如"产品"）TF-IDF 低，影响小。

#### 优点与缺点

| 优点 | 缺点 |
|------|------|
| 训练极快（秒级），不需要 GPU | 特征独立假设过强，忽视词序和上下文 |
| 可解释性强 | 对长文本效果不佳 |
| 天然适合高维稀疏特征 | 只能拟合线性决策边界 |

---

### 2. LinearSVC（线性支持向量机）

#### 是什么

线性支持向量机的目标是：**在 5000 维的 TF-IDF 空间中，找一条最优的超平面把差评和好评的样本点分隔开**。"最优"有两层含义：

1. **最大化分类间隔（margin）**：超平面到离它最近的样本（支持向量）的距离要尽可能大。间隔越大，对新样本的容错能力越强。
2. **允许少量犯错（软间隔）**：现实中数据不可能是完全线可分的（总有模棱两可的评论），通过参数 C 控制对错误的容忍度。

一条评论在 TF-IDF 空间中被表示为 5000 维空间中的一个点。LinearSVC 学到一个 5000 维的权重向量 w：正权重对应"好评倾向词"（如"满意""很好"），负权重对应"差评倾向词"（如"太差了""垃圾"），接近 0 的权重对应中性词（如"产品""的"）。

#### 为什么用在这个项目

- **高维稀疏数据上表现天然稳定**：TF-IDF 空间的 5000 维中，大多数词对情感没贡献，`(1/2)||w||²` 正则化会天然惩罚过大的权重，迫使模型"把力气花在刀刃上"，只给真正有情感区分力的词分配显著权重。这等价于隐式的特征选择。
- **比 NB 更不"朴素"**：SVM 不假设特征独立，它从全局最优的角度同时考虑所有词的权重组合。
- **收敛性好**：只有一个核心超参 C，调参成本极低。

#### 如何训练的

**目标函数**（LinearSVC 最小化以下式子）：

```
min (1/2)||w||² + C × Σ max(0, 1 - yᵢ(w·xᵢ + b))
    └─正则化项──┘   └──────── Hinge Loss 累加 ────────┘
    "最大化间隔"     "惩罚分类错误"
```

- `(1/2)||w||²` = 权重向量 w 的 L2 范数平方。最小化它等价于最大化 margin（间隔），迫使模型用尽可能小的权重来做判断。
- `Σ max(0, 1 - y(wx+b))` = Hinge Loss（合页损失）。如果样本被正确分类且离边界足够远，该项为 0；否则按"偏离程度"线性惩罚。
- `C = 1.0` 平衡两个目标。C 越大 → 更关注减少训练错误（可能过拟合）；C 越小 → 更关注最大化间隔（可能欠拟合）。

SVM 不在乎"预测对了多少分"，只关心"有没有分对边"——样本只要落在自己类别那侧的超平面之外就算好。

**参数选择**：

| 参数 | 值 | 原因 |
|------|---|------|
| `C=1.0` | 默认平衡值 | 在 5000 维 TF-IDF 上表现稳定 |
| `max_iter=2000` | 比默认 1000 大 | 确保在高维稀疏数据上充分收敛 |
| `dual=False` | 用 primal 优化 | 样本数 > 特征数(5000) 时 primal 更快 |

**为什么外层包了 CalibratedClassifierCV**：LinearSVC 本身只能输出决策值（正负），不能输出概率。CalibratedClassifierCV 通过 **Platt Scaling**（用 S 型 sigmoid 函数将决策值映射到 [0,1] 区间），让 SVM 也能输出 `[0.12, 0.88]` 格式的概率，与 NB、XGBoost 的接口统一。

#### 优点与缺点

| 优点 | 缺点 |
|------|------|
| 高维稀疏数据上天然擅长，收敛稳定 | 线性决策边界，无法捕捉非线性语义关系 |
| 只有一个核心超参 C，几乎不用调 | 需要额外 Platt Scaling 才能输出概率 |
| 正则化天然抗过拟合 | 数据量增大时训练变慢 |

### 3. XGBoost（极端梯度提升）

#### 什么是决策树（XGBoost 的基本单元）

在讲 XGBoost 之前，先理解它的基本构建块——**决策树**。决策树就是一连串 if-else 规则，像流水线一样把样本一层层筛下去：

```
             TF-IDF("坏了") > 0.3 ?
                /            \
              是              否
              │               │
        TF-IDF("满意") > 0.2?    TF-IDF("不错") > 0.3?
          /        \              /        \
        是         否           是         否
        │           │           │           │
     【好评】  TF-IDF("差")>0.1? 【好评】  【差评】
                /      \
              是       否
              │         │
          【差评】    【好评】
```

在每个节点，算法会选一个特征和一个阈值把样本分成两堆——选的标准是"分完后两堆尽量纯"（一边尽可能全是差评，另一边尽可能全是好评）。一直分裂直到满足停止条件，最后每个叶子节点给出一个分数。

#### 什么是 XGBoost

XGBoost（eXtreme Gradient Boosting）是一种**梯度提升树**算法。它的核心思路很朴素：一棵小决策树不够强，那就用**很多棵小树接力做判断**。每棵新树不重新学习整个分类任务，而是专门去修正前面所有树加在一起的"残差"（预测值和真实值之间的差距）。

打个比方：第一棵树大致判断"整体偏向好评"，残差是一部分差评还没被识别出来；第二棵树就专门学"前面判错了哪些情况"；第三棵再补第二棵的漏洞……一棵接一棵地叠加（200 棵），最终把误差压到很低。同时单棵树很浅（`max_depth=6`）又保证了每棵不会死记硬背，泛化能力好。

**和随机森林的区别**：

| | 随机森林 | XGBoost |
|---|---|---|
| 树的关系 | **并行**——各建各的，互不依赖，最后投票 | **串联**——后一棵修正前一棵的残差 |
| 每棵树做什么 | 各自独立学好整个分类任务 | 只关注前面树没做好的那部分 |
| 最终预测 | 所有树投票取多数 | 所有树输出依次累加（加权） |
| 优化方式 | 不归路——每棵树建好就定型了 | 用梯度信息指导每棵树往哪个方向建 |

#### 为什么用在这个项目

- **可处理非线性关系**：决策树天然处理特征交互，能学到"坏了" AND "客服差" 同时出现时的组合模式
- **字符级特征补强**：调优模式下加入 char 级 TF-IDF（1/2/3-gram），不依赖 jieba 分词精度即可捕获字面模式
- **机器学习中最强单模型之一**：在有足够数据时通常显著优于 NB 和 SVM

#### 如何训练的

**默认模式**：输入为 5000 维词级 TF-IDF，手动设定核心超参：

| 参数 | 值 | 含义 |
|------|---|------|
| `n_estimators=200` | 200 棵树 | 迭代 200 轮，每轮加一棵新树 |
| `max_depth=6` | 树深 6 | 单棵树最多 2⁶=64 个叶子，控制复杂度 |
| `learning_rate=0.1` | 学习率 0.1 | 每棵树只贡献最终预测的 10%，剩下 90% 留给后续树修正 |

**调优模式**（`--tune-xgboost`）：特征从 5000 维升级为 **10000 维组合特征**：

- 词级 TF-IDF 5000 维：jieba 分词 → unigram+bigram
- 字级 TF-IDF 5000 维：直接在原始文本上提取 1/2/3-gram（`analyzer="char"`）

两种特征用 `scipy.sparse.hstack` 水平拼接。字级特征的加入让模型即使面对未登录词（jieba 切不出来的新词/错别字）也能从字形层面捕获模式。

超参数搜索策略：从训练集中分层抽样 15000 条 → `RandomizedSearchCV`（20 组随机参数 × 3 折交叉验证，以 macro-F1 为评分指标）→ 找到最佳参数后在**全量数据**上重新训练最终模型。

搜索的超参数空间：

| 参数 | 搜索范围 | 含义 |
|------|---------|------|
| `n_estimators` | [200, 300, 500] | 树的数量 |
| `max_depth` | [6, 8, 10] | 树的最大深度 |
| `learning_rate` | [0.05, 0.1] | 学习率 |
| `subsample` | [0.7, 0.8, 1.0] | 每棵树用多大比例的训练样本（行采样） |
| `colsample_bytree` | [0.6, 0.8, 1.0] | 每棵树用多大比例的特征（列采样） |
| `reg_alpha` | [0, 0.1, 1.0] | L1 正则化系数 |
| `reg_lambda` | [1.0, 5.0] | L2 正则化系数 |

**训练时的内部机制**：

1. 初始预测全部为 0.5（log odds = 0，正负概率各半）
2. 第 1 棵树：计算每个样本的一阶和二阶梯度（残差的方向和大小），找一个最优的分裂特征和阈值，把样本分成两堆使梯度下降最快。不断递归分裂直到 `max_depth=6` 或样本太少
3. 第 2 棵树：重新计算残差（经过第 1 棵修正后还剩的误差），再建一棵新树专门修正这些残差
4. 重复 200 轮
5. 最终预测 = 200 棵树的输出加权求和

**预测新评论**：

```
logit = 树₁输出 × 0.1 + 树₂输出 × 0.1 + ... + 树₂₀₀输出 × 0.1
```

每条树根据评论的 TF-IDF 特征值沿决策路径走到某个叶子节点，输出该叶子节点的权重（正=好评倾向，负=差评倾向）。`learning_rate=0.1` 让每棵树只做微小调整，避免一棵树带偏全局。

#### 优点与缺点

| 优点 | 缺点 |
|------|------|
| 可处理非线性关系，内置缺失值处理 | 对高维稀疏矩阵的表现不如神经网络 |
| 字符级组合特征弥补了词级 TF-IDF 的信息损失 | 调参成本高，搜索空间大 |
| 通常比 NB 和 SVM 有明显提升 | 训练较慢，尤其在组合特征模式下 |

### 4. TextCNN（文本卷积神经网络）

#### 是什么

TextCNN 把文本当作一维"图像"来处理。在图像中，卷积核（比如 3×3 像素）在图片上滑动检测局部模式（边缘、纹理）；在文本中，不同尺寸的卷积核（3/4/5-gram）在词向量序列上滑动检测**不同粒度的局部短语模式**。

**关键操作**：

1. **卷积**：每个卷积核是一个可学习的"模式检测器"。比如某个 3-gram 卷积核可能学会了检测"很 + 好/满意/不错"这类正向短语模式，另一个可能学会了检测"太 + 差/失望/烂"这类负向短语模式。卷积核在 128 个词向量序列上从上到下滑动，每个位置输出一个匹配程度得分。

2. **MaxPool（最大池化）**：对每个卷积核的所有滑动位置的得分取最大值。这个操作的核心思想是——"不管关键短语出现在句子的第 1 个位置还是第 100 个位置，只要出现过就判定为有"。这天然解决了情感信号位置不确定的问题。

3. **多尺寸组合**：3-gram 捕捉短语级（"很不错"、"太差了"），4-gram 捕捉短句级（"质量非常好"），5-gram 捕捉更长模式。三种尺寸的输出拼接后送入全连接层，让模型综合考虑不同粒度的语义。

#### 为什么用在这个项目

中文情感分类的情感信号高度集中在**局部短语**中而非均匀分布在全句：

| 评论 | 局部信号 |
|------|---------|
| "产品质量**很好**，做工**精细**" | 正向短语集中在局部 |
| "外观不错，但用了几天**就坏了**" | 转折后的负面短语才是关键 |
| "**太失望了**，完全不行" | 情感词连续出现 |

如果简单对整句做平均（如 FastText），正面和负面信号会互相抵消。TextCNN 的多尺寸卷积核 + MaxPool 能精准定位这些局部关键信号。

#### 如何训练的

**词向量**：每个词被映射为一个 300 维的向量。可以使用随机初始化（正态分布），也可以加载预训练的中文词向量（推荐，能覆盖约 70-80% 的词汇，大幅提升效果）。

**特征提取流程**（以 "产品质量很好" 为例）：

| 步骤 | 操作 | 输出形状 | 说明 |
|------|------|---------|------|
| ① | Embedding 查表 | (1, 128, 300) | 128 个词的 300 维向量序列 |
| ② | 3-gram Conv2d(1→100, kernel=3×300) | (1, 100, 126, 1) | 100 个卷积核，每个在 128 词上滑出 126 个位置 |
| ③ | MaxPool 取最大值 | (1, 100, 1) | 每个卷积核保留全句最强信号 |
| ④ | 4-gram/5-gram 同理 | (1, 100) × 3 | — |
| ⑤ | 三个尺寸输出拼接 | (1, 300) | 300 维特征向量 |
| ⑥ | Dropout + Linear(300→2) | (1, 2) | 分类 logits |

**训练配置**：

- 优化器：**Adam**，初始学习率 1e-3
- 学习率调度：**ReduceLROnPlateau** — 当验证集损失连续 2 个 epoch 不下降时，学习率自动减半
- **EarlyStopping**（patience=5）：监控验证集损失，连续 5 个 epoch 没有改善就停止。用 val_loss 而非 val_f1 判断（因为 loss 连续平滑，F1 波动大容易误判），但最终保存的是 val_f1 最高的模型
- 正则化：**Dropout=0.5**，训练时随机丢弃 50% 的神经元，防止过拟合
- 最大训练 30 个 epoch

#### 优点与缺点

| 优点 | 缺点 |
|------|------|
| 结构简单，训练快，参数量小 | 卷积核视野有限（最多 5-gram），无法捕捉长距离依赖 |
| 对局部短语模式敏感，适合中文短评 | 对需要全局上下文理解的长句（如"虽然...但是..."的远距离转折）能力不足 |
| 可配合预训练词向量快速提升效果 | 需要自行构建词表，未登录词只能用 UNK 代替 |

### 5. BiGRU-Attention（双向 GRU + 注意力机制）

#### 什么是 GRU

GRU（Gated Recurrent Unit，门控循环单元）是一种**循环神经网络（RNN）**。RNN 的核心思想是：按顺序读入句子中的每个词，每读一个词就更新自己的"记忆状态"。GRU 有两种门控：

- **重置门**：决定"多少旧信息该忘掉"
- **更新门**：决定"多少新信息该写入记忆"

为什么用 GRU 不用 LSTM？GRU 只有 2 个门（LSTM 有 3 个），参数更少、训练更快，在短文本分类任务上效果相当。

**为什么用双向？** 单向只能用到上文信息。比如读"不"这个字时，单向模型不知道后面是"好"还是"差"——"不好"和"不差"是截然相反的语义。双向 GRU 同时从前向后和从后向前各跑一遍，每个位置的输出同时包含了上下文。

#### 什么是注意力机制

这里使用的是 **Bahdanau 加法注意力（Additive Attention）**。

**问题**：BiGRU 虽然每个时间步都有输出，但最终做分类时如果只看最后一步的隐藏状态，序列早期的重要信息（比如句首的"非常满意"）会被逐渐"遗忘"。

**解决**：注意力机制不只看最后一步，而是对**所有时间步**的 GRU 输出计算一个"重要性得分"，然后加权求和。模型会自动把注意力集中在关键的情感词上：

```
"虽然 包装 一般 ，但 产品 质量 非常 好"
  ↓
 [0.01, 0.02, 0.03, 0.01, 0.05, 0.10, 0.35, 0.43]  ← 注意力权重
                                        ↑       ↑
                                     "非常"  "好" 权重最高
```

**注意力分数的计算**（加法注意力）：

```
score(h_t) = v^T · tanh(W · h_t)
```

- `W`：一个可学习的线性变换矩阵，把 GRU 输出映射到新的隐藏空间
- `tanh`：非线性激活，让模型能学习更复杂的打分逻辑
- `v`：一个可学习的向量，把变换后的表示压缩为标量得分
- 最后对所有得分做 **softmax** 归一化

直观理解：`W` 相当于"重新理解"每个词在当前上下文的含义，`v` 相当于提问"这个词对判断情感有多重要？"

#### 如何训练的

**网络结构**：

```
Embedding(30000×300) → BiGRU(300→128, 双向=×2)
                     → Attention(hidden_dim=256) → 加权上下文向量(256维)
                     → Dropout(0.5) → Linear(256→2)
```

**完整数据流**：

| 步骤 | 操作 | 输出形状 |
|------|------|---------|
| ① | jieba 分词 → 词表索引 → 截断/补齐 | (B, 128) |
| ② | Embedding 查表 | (B, 128, 300) |
| ③ | BiGRU 双向编码 | (B, 128, 256) |
| ④ | Attention 计算权重 + 加权求和 | (B, 256) |
| ⑤ | Dropout + Linear 分类 | (B, 2) |

**训练配置**：与 TextCNN 完全相同——Adam(lr=1e-3)、ReduceLROnPlateau、EarlyStopping(patience=5)、Dropout=0.5、最大 30 epochs。

#### 优点与缺点

| 优点 | 缺点 |
|------|------|
| 可捕捉长距离上下文依赖 | 序列模型无法并行，训练比 TextCNN 慢 |
| 注意力权重可解释——可视化为模型关注的关键词 | 参数量比 TextCNN 大 |
| 通常比 TextCNN 高出 1-2 个 F1 点 | 对短文本的优势不明显 |

### 6. BERT-base-Chinese

#### 什么是 BERT

BERT（Bidirectional Encoder Representations from Transformers）是 Google 在 2018 年提出的预训练语言模型，它彻底改变了 NLP 领域的范式。

**核心创新**：在此之前，NLP 模型通常是"从零开始"训练的（比如前面的 TextCNN 和 BiGRU），能用的数据只有手头标注好的几千到几万条。BERT 的思路是——**把"学习语言本身"和"学习具体任务"分开**：

1. **预训练阶段**（Google 已经做好了）：用海量无标注的中文文本（维基百科 + 新闻 + 百科等），以两个自监督任务训练：
   - **MLM（掩码语言模型）**：随机遮住 15% 的词让模型猜。比如"产品[遮住]很好"→ 模型需要从上下文推断出被遮的是"质量"。这迫使模型学会深层语义理解。
   - **NSP（下一句预测）**：判断两句话是否相邻，教会模型理解句子间关系。

2. **微调阶段**（我们做的事）：在 [CLS] token（BERT 在句首加的特殊标记，聚合了整句的语义信息）的最后一层输出上接一个简单的 2 分类线性头，用我们的评论数据做少量训练（仅 5 个 epoch），微调所有参数。

**为什么说 BERT 是"通才"模型**：经过预训练，BERT 已经学会了中文的语法、语义、常识等。我们来微调时，不需要从头教它中文——只需要告诉它"这个具体任务的判断标准是什么"即可。这也是为什么微调 epoch 很少（5 vs TextCNN 的 30）——它不是从零学，而是"适应"。

#### 为什么用在这个项目

- **语义理解能力远超从零训练的模型**：预训练带来的知识是几千条标注数据无法比拟的
- **开箱即用**：transformers 库已封装好加载和训练流程
- **作为预训练模型的代表**：和 RoBERTa 一起验证"通才模型+微调"在电商评论场景下的天花板

#### 如何训练的

**模型信息**：

| 属性 | 值 |
|------|---|
| 参数量 | ~110M（1.1 亿） |
| 层数 | 12 层 Transformer 编码器 |
| 隐藏维度 | 768 |
| 注意力头 | 12 个 |
| 词汇表 | 21,128 个中文字符级 token |
| 最大输入长度 | 256 tokens |

**训练配置**（与从零训练的关键区别）：

| 参数 | 值 | 为什么不同 |
|------|---|----------|
| 学习率 | **2e-5** | 远小于从零训练的 1e-3。预训练权重要小心保护，不能大步更新 |
| 批大小 | 16 | GPU 显存限制（110M 参数 + 优化器状态） |
| 最大 epoch | 5 | 微调过久容易过拟合（模型已经很强了，很快就能适应新任务） |
| 优化器 | AdamW | Adam + 解耦的 weight decay（权重衰减 0.01），是 Transformer 训练的标准配置 |
| LR 调度 | Warmup + 线性衰减 | 前 10% 步数学习率从 0 线性升到 2e-5（warmup），之后线性衰减到 0。warmup 防止训练初期梯度震荡 |
| 精度 | FP16 混合精度 | GPU 可用时自动启用，显存减半、速度加倍 |
| EarlyStopping | patience=2 | 比从零训练的 5 更激进——预训练模型适应很快，2 个 epoch 没提升就是真的到瓶颈了 |

**模型缓存**：脚本优先从本地 HuggingFace 缓存加载模型（`local_files_only=True`），失败后自动从 HuggingFace Hub 下载。首次运行需要网络和较长的下载时间（约 400MB），后续运行直接使用缓存，无需网络。

#### 优点与缺点

| 优点 | 缺点 |
|------|------|
| 语义理解能力极强，效果远超从零训练模型 | 推理慢，参数量大（110M vs TextCNN 的 ~10M） |
| 开箱即用，无需预处理（自带 tokenizer） | 显存占用高，需要较大 GPU |
| 中文 BERT 预训练质量高 | 不适合实时高并发场景 |

### 7. RoBERTa-wwm-ext

#### 是什么

RoBERTa（Robustly optimized BERT approach）是 BERT 的改进版，由 Facebook AI 在 2019 年提出。本项目的具体版本 `hfl/chinese-roberta-wwm-ext` 由**哈工大讯飞联合实验室**发布。

**RoBERTa 对 BERT 的改进**：

1. **取消 NSP 任务**：研究发现 BERT 的"下一句预测"任务对下游任务帮助不大，反而限制了模型。取消后用更大的 batch 和更多数据训练，效果更好。
2. **动态掩码**：BERT 的 MLM 是在预处理时一次性生成固定的掩码，每个 epoch 看到的掩码都一样。RoBERTa 每 epoch 重新随机生成掩码，数据利用率更高。
3. **更多数据、更长训练**：比 BERT 训练数据更大、步数更多。

**WWM（Whole Word Masking，全词遮蔽）**：BERT 的中文版本以**单字**为单位做掩码，比如"产品很好"会随机遮住"产"或"品"等单字。WWM 改为以**整词**为单位——如果"产品"这个词的一部分被选中，整个词一起被遮住。这迫使模型学习更完整的语义信息，而非依赖单字字形猜测。

**Ext（Extended data）**：在比原始 RoBERTa 更大的中文语料（CLUECorpusSmall + 百科 + 问答等）上训练，覆盖更广。

#### 为什么用在这个项目

- 是本项目中**效果最好的模型**（Macro-F1 0.9075）
- WWM 在中文场景下特别有用——中文的最小语义单元是词而非字，整词遮蔽更符合中文的语义结构
- 作为 BERT 的对照，验证改进策略在我们的评论数据上是否有效（答案：是的，约高 0.5 个 F1 点）

#### 如何训练的

训练配置与 BERT 完全相同（同样的参数量、学习率、epoch、所有超参）。唯一区别是模型路径：

```python
ROBERTA_MODEL_NAME = "hfl/chinese-roberta-wwm-ext"
```

这意味着两个模型性能差异**仅来自预训练策略的差异**，而非训练配置的不同——是一个公平的对比。

#### 优点与缺点

| 优点 | 缺点 |
|------|------|
| 中文场景下效果最优，WWM 设计更贴合中文特点 | 与 BERT 完全相同的推理开销 |
| 通常比原生 BERT 高 0.5-1 个 F1 点 | 首次下载约 400MB+，需要网络 |
| 哈工大出品，社区维护活跃，文档齐全 | 对于极短评论（几个词），优势不明显 |

## 实验结果对比

所有模型在测试集上的表现（按 Macro-F1 降序排列）：

| 模型 | Macro-F1 | Accuracy | 差评 F1 | 好评 F1 |
|------|----------|----------|---------|---------|
| RoBERTa-wwm-ext | **0.9075** | 0.9075 | 0.9075 | 0.9076 |
| BERT-base-Chinese | 0.9026 | 0.9026 | 0.9027 | 0.9025 |
| BiGRU-Attention | 0.8903 | 0.8903 | 0.8895 | 0.8911 |
| TextCNN | 0.8808 | 0.8808 | 0.8827 | 0.8788 |
| XGBoost | 0.8767 | 0.8767 | 0.8758 | 0.8776 |
| LinearSVC | 0.8477 | 0.8477 | 0.8453 | 0.8501 |
| MultinomialNB | 0.8387 | 0.8387 | 0.8346 | 0.8427 |

**关键观察**：

1. **预训练模型 > DL 从零训练 > 传统 ML**：从 0.8387 到 0.9075，跨度约 7 个 F1 点。越复杂的模型通常效果越好，但边际收益递减。
2. **BERT vs RoBERTa**：WWM + 动态掩码 + 更多数据带来了 0.5 个 F1 的提升，证实了预训练策略改进的有效性。
3. **BiGRU-Attn vs TextCNN**：注意力机制 + 双向上下文让 BiGRU 比纯卷积的 TextCNN 高出约 1 个 F1 点。
4. **XGBoost vs 线性模型**：非线性集成方法比线性 SVM 高了约 3 个 F1 点。字符级+词级组合特征的加入功不可没。
5. **差评 F1 vs 好评 F1**：所有模型在两类上的 F1 非常接近（差评最大 ±0.01），说明均衡策略成功避免了类别偏向。

## 技术要点

- **预分词**：`TokenizedDataset` 在 `__init__` 中一次性完成所有文本的 jieba 分词和索引转换，避免每个 epoch 都重复运行 `jieba.cut()`，训练速度大幅提升。
- **模型缓存**：预训练模型使用 `local_files_only=True` 优先加载本地缓存，首次下载后无需网络，避免 HuggingFace Hub 超时。
- **无头服务器兼容**：`matplotlib` 使用 `Agg` 后端（非交互式渲染），可在无 GUI 的 GPU 服务器上正常保存图表。
- **脚本独立性**：每个训练脚本是独立入口点，共享 `src/` 中的评估和可视化模块，但各自拥有独立的模型定义和训练逻辑。
- **统一推理接口**：`app/model_loader.py` 的 `SentimentPredictor` 类为所有 7 种模型提供统一的 `predict(text) → (label, probs)` 接口，前端无需关心底层模型类型。
- **两种前端**：Gradio 版 (`demo.py`) 快速搭建，开箱即用；FastAPI 桌面版 (`server.py` + `static/index.html`) 提供桌面优化的双栏布局，左右分区展示输入控件与分析结果。
- **延迟加载**：两个版本的 Web Demo 均采用懒加载策略——仅当用户首次切换到某模型时才加载权重，降低内存占用和启动时间。

## 扩展方向

- **新增预训练模型**：在 [04_train_pretrained.py](scripts/04_train_pretrained.py) 的 `main()` 中添加新分支（如 ERNIE、ELECTRA），在 `config.py` 中添加模型名称常量。
- **新增 DL 模型**：在 `src/models/` 中创建新文件（如 `lstm.py`），在 [03_train_dl.py](scripts/03_train_dl.py) 中注册即可。
- **多分类扩展**：修改 `star_to_label()` 为三分类，调整各脚本的 `num_classes` 参数。
- **模型集成**：组合多个模型的预测结果（投票/加权），通常能进一步提升准确率。
- **数据增强**：通过回译、同义词替换等方式扩充训练数据，提升小样本下的泛化能力。
